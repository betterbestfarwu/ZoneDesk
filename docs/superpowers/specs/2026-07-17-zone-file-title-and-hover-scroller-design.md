# 分区文件名布局与悬停滚动条设计

## 目标

在现有 Finder 风格分区文件网格上完善两个细节：文件名按照可读性规则在两行换行或切换为单行中间省略；图标选择框与文件名标签保持严格 6px 间距；滚动条仅在鼠标位于分区窗口内且内容确实溢出时显示。

## 范围

- 文件名能在两行内完整显示时，自动按字符换行并显示一至两行。
- 文件名完整显示需要超过两行时，改为单行显示，并在文字中间使用 `...`。
- 中间省略同时保留文件名前缀和扩展名侧的末尾信息。
- 选中与未选中文件使用同一份文本布局结果。
- 选中时蓝色文件名标签按最终一行或两行文字区域绘制。
- 图标选择框底边与蓝色文件名标签顶边保持严格 6px 垂直间距。
- 分区内容溢出时，鼠标进入整个分区窗口后显示滚动条。
- 鼠标移出整个分区窗口后立即隐藏滚动条。
- 内容没有溢出或正在编辑分区布局时，滚动条保持隐藏。
- 文件刷新和分区窗口尺寸变化后重新判断是否溢出。

## 不在范围内

- 改变文件图标尺寸、Finder 网格同步规则或文件选择模型。
- Command 多选、Shift 连选、框选、键盘导航或文件拖放。
- 文件名编辑、悬停工具提示或完整路径展示。
- 滚动条淡入淡出动画。
- 修复现有透明滚动条点击测试或改变滚动计算逻辑。

## 方案选择

### 文件名布局

采用 AppKit TextKit 进行测量和绘制，不继续依赖 `NSString.draw(in:)` 隐式处理多行截断。

TextKit 布局单元包含 `NSTextStorage`、`NSLayoutManager` 和无内部 padding 的 `NSTextContainer`。布局过程分两步：

1. 先使用文件单元格可用宽度和最多两行的高度测量完整文件名。
2. 如果完整文本的字形范围能在两行内结束，则保留一至两行自动换行结果；如果仍有未布局字形，则重新布局为单行并使用 `.byTruncatingMiddle`。

两种结果都输出同一结构：最终绘制矩形、实际文字区域、一行或两行的行数，以及是否发生中间省略。`ZoneFilesView` 使用该结构同时绘制文字和选中背景，避免背景与文字采用不同测量方式。

没有选择将 `NSTextField` 作为每个文件的子视图，因为它会改变现有自绘网格、命中测试和滚动结构；也没有继续使用 `NSString.draw`，因为它无法可靠判断完整名称是否已经超过两行。

### 6px 间距

当前图标选择框在实际图标矩形四周外扩 4px。文件名布局的垂直起点不再直接从图标矩形底部计算，而是使用：

```text
titleBackground.minY = iconSelectionRect.maxY + 6
```

选中文字和未选中文字使用相同的标题起点。选中状态下，该起点是蓝色标签顶边；未选中状态下，它是文字布局区域顶边。测试直接比较两个矩形边界，禁止使用近似值。

如果当前 Finder 网格单元格高度不足以容纳图标、6px 间距和最终标题高度，单元格派生高度取三者总高度与 Finder 网格尺寸中的较大值，避免与下一行文件重叠。

### 滚动条显隐

由 `ZoneView` 管理整个分区的鼠标状态，不在文件网格或滚动条自身上建立局部显隐规则。

- `ZoneView` 使用 `.activeAlways`、`.inVisibleRect` 和 `.mouseEnteredAndExited` 建立跟踪区域。
- `mouseEntered` 将 `isPointerInside` 设为 `true`。
- `mouseExited` 将 `isPointerInside` 设为 `false`。
- `filesScrollView.autohidesScrollers` 设为 `false`，避免 AppKit 默认逻辑覆盖显式状态。
- `updateScrollerVisibility()` 是唯一修改垂直滚动条 `isHidden` 的入口。

滚动条显示条件为：

```text
isPointerInside
&& !isEditing
&& verticalScroller.isEnabled
&& documentHeight > viewportHeight
```

`updateScrollerVisibility()` 在鼠标进入、鼠标移出、文件刷新、分区编辑状态变化以及 `ZoneView.layout()` 完成滚动范围计算后调用。显示和隐藏立即生效，不增加动画或延迟。

## 数据流

### 文件名

1. `ZoneFilesView.rebuildCells()` 根据 Finder 布局计算图标和单元格矩形。
2. 文件名布局器尝试在两行内完整布局文件名。
3. 两行能完整显示时返回多行结果；否则返回单行中间省略结果。
4. 图标选择框确定后，标题起点设置为其底边加 6px。
5. `draw(_:)` 使用同一结果绘制蓝色标签和文字。

### 滚动条

1. 文件或窗口尺寸变化更新文档视图高度和 scroller enabled 状态。
2. `ZoneView` 更新鼠标是否位于分区内。
3. `updateScrollerVisibility()` 合并鼠标、编辑状态和溢出状态。
4. 只有四项条件全部满足时显示滚动条，其余情况隐藏。

## 测试

### 文件名布局测试

- 短文件名完整显示为一行，不省略。
- 较长但能容纳的文件名自动换行为两行，不省略。
- 完整显示需要超过两行时切换为单行。
- 单行结果使用中间 `...`，同时保留名称前缀和末尾扩展名。
- 选中与未选中状态返回相同的文字行数和文本矩形。
- 蓝色标签顶边减去图标选择框底边严格等于 6px。
- Finder 字体尺寸变化后仍遵守一行/两行/单行中间省略规则。

### 滚动条测试

- 初始状态隐藏滚动条。
- 内容溢出且鼠标进入分区时显示滚动条。
- 鼠标移出分区后隐藏滚动条。
- 内容未溢出时，鼠标进入仍保持隐藏。
- 编辑分区布局时，鼠标进入仍保持隐藏。
- 内容从不溢出变为溢出后，在鼠标仍位于分区内时显示。
- 分区尺寸变大使内容不再溢出时立即隐藏。

## 兼容性与风险

- 项目最低支持 macOS 12，所用 TextKit 1 和 `NSTextContainer.maximumNumberOfLines` API 均在该版本可用。
- 超过两行后会刻意切换为单行，因此超长名称的标签高度会从两行变为一行；这是已确认的产品规则。
- 中间省略由 TextKit 决定具体省略字符位置，不手工截断 Unicode 字符，避免拆开组合字符或 emoji。
- 显式隐藏滚动条后必须在布局完成后再读取 `isEnabled` 和文档高度，否则可能使用上一次布局状态。
- 现有三个透明滚动条点击测试在本次设计前已失败；本设计不通过禁用测试掩盖它们，也不把它们计入本功能完成条件。
