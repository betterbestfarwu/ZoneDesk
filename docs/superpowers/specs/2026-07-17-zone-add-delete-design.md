# 分区新增与安全删除设计

## 目标

在现有“编辑分区布局”模式中增加新增和删除分区功能。新增分区是空白自定义容器，不参与桌面文件自动分类；删除分区时，将其中的文件安全移回桌面，确认没有文件遗留后再删除分区目录和配置。

## 范围

本次包含：

- 编辑模式菜单中的“新增分区…”操作。
- 编辑模式菜单中的“删除当前分区…”操作。
- 新增分区名称输入、默认位置和默认尺寸。
- 删除确认、文件移回桌面、同名避让及失败保护。
- 分区增删的配置持久化。
- 防止用户删除的默认分区在下次启动时被自动补回。

本次不包含：

- 为新分区选择自动分类文件类型。
- 在分区窗口内增加加号或删除按钮。
- 分区之间拖放文件。
- 批量删除多个分区。

## 交互设计

进入“编辑分区布局”后，状态栏菜单在现有“重命名当前分区…”附近显示：

- “新增分区…”：始终可用。
- “删除当前分区…”：仅在已选中分区时可用。

新增操作显示名称输入框，默认名称为“新分区”。名称去除首尾空白后不能为空。创建成功后，新分区立即显示并写入配置。新分区使用现有分区默认尺寸 `300 × 220`，放置在主屏幕可见范围内；若默认位置与已有分区重叠，则按固定偏移寻找下一个仍在屏幕内的位置。新分区的 `acceptedCategories` 为空，因此不会接收自动归纳的桌面文件。

分区目录按名称生成。若名称映射出的目录已经存在，新增操作必须失败并提示用户更换名称，不得把新分区静默接入已有目录。

删除操作需要先选中分区。确认框明确显示分区名称，并说明分区内文件将移回桌面。用户取消时不修改文件、目录或配置。

## 文件与配置流程

### 新增分区

1. App 层取得并校验名称。
2. 创建具有新 UUID、默认矩形、空 `acceptedCategories`、`locked = false` 的 `ZoneModel`。
3. 由 `AppConfig` 的新增接口写入内存配置。
4. 创建对应收纳库目录。
5. 目录创建成功后保存配置、刷新窗口和文件列表。
6. 目录创建失败时不写入配置，并记录可排查错误。

### 删除分区

1. App 层取得当前选中分区并请求用户确认。
2. `ZoneLibrary` 枚举分区目录中的全部非隐藏项目，并逐项移回桌面。
3. 桌面已有同名项目时，使用现有的稳定编号规则生成 `名称 2.ext`、`名称 3.ext`，绝不覆盖。
4. 所有项目移动成功后删除空的分区目录。
5. 只有文件迁移和目录删除都成功后，才从 `AppConfig` 删除分区并保存配置。
6. 刷新所有分区窗口和状态栏菜单。

隐藏项目也属于用户数据。删除分区时不能静默丢弃隐藏项目：如果目录仍包含未迁移项目，目录删除会失败，分区配置必须保留，并向用户显示删除未完成的错误。

## 默认分区迁移兼容

当前 `ConfigManager.load` 会在每次加载时补齐缺少文件分类的默认分区，导致用户删除默认分区后它在重启时重新出现。

配置增加可选的默认分区迁移版本字段。规则如下：

- 新生成的配置直接标记为已完成当前迁移。
- 缺少该字段的旧配置在加载时执行一次现有“补齐缺失默认分类”逻辑，并在后续保存时写入当前迁移版本。
- 已带当前迁移版本的配置按用户保存内容原样加载，不再根据缺失分类补回分区。

这样既保留旧配置兼容，也让之后的删除永久生效。

## 模块边界

### `AppConfig`

负责纯数据变更：

- 新增一个 ID 唯一的分区。
- 按 ID 删除分区并返回是否成功。
- 保存默认分区迁移版本。

### `ZoneLibrary`

负责磁盘操作：

- 将指定分区目录内容移回目标桌面目录。
- 复用统一的同名避让规则。
- 返回已移动项目和失败信息，便于 App 层决定是否删除配置。
- 仅在目录真正为空时删除分区目录。

### AppKit 应用层

负责菜单状态、名称输入、删除确认、错误提示、调用核心接口以及刷新窗口，不自行拼接文件路径或执行文件移动。

## 错误处理

- 新增目录失败：不新增配置，显示错误，现有分区不受影响。
- 单个文件移动失败：停止最终删除，保留分区配置和目录，已成功移回桌面的文件不再移回收纳库。
- 分区目录不存在：视为空分区，可以继续删除配置。
- 删除空目录失败：保留分区配置并显示错误。
- 配置保存失败：记录明确错误；磁盘文件不得因此被覆盖或删除。
- 日志可以包含本机文件路径以便排查，但不得记录 token、cookie、密码或其他凭据。

## 测试

核心自动化测试：

- `AppConfig` 可以新增空白分区，并拒绝重复 ID。
- `AppConfig` 可以按 ID 删除分区，未知 ID 返回失败。
- 带迁移版本的配置删除默认分区后重新加载不会补回。
- 旧配置仍会执行一次默认分区补齐。
- `ZoneLibrary` 将分区文件移回桌面并删除空目录。
- 桌面同名文件存在时生成带编号的新名称且不覆盖原文件。
- 文件移动失败或目录仍非空时报告失败并保留目录。

应用层最小验证：

- 编辑模式显示新增和删除菜单项，普通模式不显示。
- 未选中分区时删除菜单不可用。
- 新增后窗口立即出现，重启后仍存在。
- 删除确认框显示正确分区名称。
- 删除后文件出现在桌面，分区立即消失且重启后不恢复。

完整验证命令：

```bash
swift test
swift build
```

## 约束

- 保持 Swift 5.9 和 macOS 12 兼容。
- 不引入第三方依赖。
- 不大范围重构现有单文件 AppKit 结构。
- 不覆盖或静默删除用户文件。
- 所有核心文件操作先由自动化测试覆盖，再接入界面。
