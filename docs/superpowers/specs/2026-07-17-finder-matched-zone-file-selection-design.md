# Finder 一致的分区文件选择与网格设计

## 目标

修正分区文件网格与 macOS Finder 桌面的差异：文件在单击后保持真实选中状态，鼠标移出不能清除；选中外观采用 Finder 风格；图标、文字和网格尺寸在应用启动及每次分区文件刷新时读取 Finder 当前桌面设置并同步。

## 范围

- 普通单击文件后仅选中该文件，并持续保留选中状态。
- 鼠标移动或移出分区不改变选择。
- 单击其他文件时切换选择。
- 单击文件网格空白区域时清除选择。
- 双击文件继续调用现有打开文件行为。
- 文件列表刷新后保留仍存在文件的选择，删除已经不存在文件的选择。
- 选中项分别绘制图标高亮和文件名高亮，不再绘制覆盖整个格子的悬停背景。
- 应用启动以及每次分区文件刷新时重新读取 Finder 的 `DesktopViewSettings.IconViewSettings`。
- 使用 Finder 当前的 `iconSize`、`gridSpacing` 和 `textSize` 计算分区文件图标、网格和文字布局。
- Finder 设置缺失、类型错误或越界时使用安全回退值。

## 不在范围内

- Command 多选、Shift 连选或框选。
- 键盘方向键导航。
- 文件拖放、重命名或右键菜单。
- 监听 Finder 偏好文件的实时变更通知；设置在下次应用启动或分区文件刷新时生效。
- 使用 Finder 私有 API 或逐像素复制 Finder 私有绘制实现。

## 设计

### Finder 布局设置

在核心层增加可测试的 Finder 桌面图标布局值类型，包含图标尺寸、网格间距和文字尺寸。解析逻辑从 `DesktopViewSettings.IconViewSettings` 读取：

- `iconSize` 控制文件图标宽高。
- `gridSpacing` 与图标尺寸共同决定单元格间距，保证分区中的文件密度随 Finder 桌面设置变化。
- `textSize` 控制文件名字体，并为最多两行文件名预留相应高度。

解析时兼容属性列表常见的整数、浮点数和 `NSNumber`。每个字段独立校验；单个字段无效时只回退该字段，不影响其他有效设置。回退值使用 Finder 常见默认配置：图标 64、网格间距 54、文字 12。

应用层维护当前布局设置。应用启动时读取一次 Finder 偏好；每次刷新分区文件前再次读取，并把最新设置传给每个 `ZoneFilesView`。视图仅消费布局值，不直接访问用户偏好，从而保持布局计算可测试。

### 文件选择模型

`ZoneFilesView` 使用文件 URL 保存当前选中项，而不是保存悬停单元格索引：

- 命中文件的普通单击将选择替换为该文件。
- 命中空白区域的普通单击清空选择。
- 鼠标移动和鼠标移出不修改选择。
- 文件列表刷新时，将选择与当前文件 URL 求交集。
- 双击先保持目标文件选中，再调用现有 `onOpenFile`。

删除 `NSTrackingArea`、`hoveredCellIndex`、`mouseMoved`、`mouseExited` 及整格 hover 绘制，避免视觉悬停被误认为真实选中。

### 选中外观

对选中文件分别绘制两个区域：

- 图标区域：围绕实际图标绘制浅色半透明圆角背景和细边框。
- 文件名区域：围绕标题实际区域绘制系统强调色圆角背景，文字使用白色。

颜色使用 AppKit 动态系统颜色，自动适配系统强调色和深浅模式。未选中文件保持透明背景和当前白色文字。

### 数据流

1. `AppDelegate` 启动或开始刷新分区文件。
2. 从 `com.apple.finder` 的 `DesktopViewSettings` 解析当前布局设置。
3. `WindowManager` 在展示或刷新窗口时把布局设置传给 `ZoneWindow`。
4. `ZoneWindow` 传给 `ZoneView`，再由 `ZoneView` 更新 `ZoneFilesView`。
5. `ZoneFilesView` 用同一布局设置计算所需高度、列数、单元格、图标和标题矩形。

## 测试

核心层测试覆盖：

- 正确解析 Finder 的 `iconSize`、`gridSpacing` 和 `textSize`。
- 兼容属性列表数值类型。
- 缺失、非数值或越界字段使用独立回退值。
- 布局设置能产生确定的单元格尺寸。

应用层测试覆盖：

- 单击文件后保持选中。
- 鼠标移出不清除选择。
- 单击另一文件替换选择。
- 单击空白区域清除选择。
- 双击仍打开正确文件。
- 刷新文件列表仅清除已不存在文件的选择。
- Finder 布局设置改变后，图标、字体、单元格和内容高度同步变化。

最终运行完整 `swift test` 和 `swift build --product zonedesk-app`。

## 兼容性与风险

- 项目最低支持 macOS 12，只使用该版本可用的公开 AppKit 和 UserDefaults API。
- Finder 没有公开其完整桌面单元格内部公式；实现以 Finder 可读的三项布局设置为唯一尺寸来源，并保持相对间距一致，不调用私有 API。
- Finder 设置刷新只发生在明确的应用启动和分区文件刷新边界，避免为偏好变化引入常驻文件监听器。
