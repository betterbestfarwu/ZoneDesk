# 分区标题栏删除按钮设计

## 目标

将编辑模式菜单中的“删除当前分区…”迁移到分区标题栏右上角。删除按钮仅在“编辑分区布局”模式显示，点击后删除按钮所属的分区。

## 范围

- 在 `ZoneView` 标题栏右上角增加无边框 AppKit 按钮。
- 按钮使用 macOS 12 可用的系统垃圾桶图标，并提供中文工具提示和辅助功能标签。
- 按钮仅在编辑模式显示；普通模式隐藏。
- 点击按钮时按分区 ID 发起删除，不依赖当前选中分区。
- 复用现有删除确认、文件移回桌面、同名避让、错误提示和配置保存流程。
- 从状态栏编辑菜单中移除“删除当前分区…”。

不包含：

- 修改“新增分区…”和“重命名当前分区…”菜单项。
- 修改删除确认文案或文件恢复规则。
- 增加无确认快速删除。

## 交互设计

进入“编辑分区布局”后，每个分区标题栏右上角显示垃圾桶按钮。按钮具有足够的点击区域，视觉尺寸保持紧凑；标题绘制区域为按钮预留空间，长标题继续使用中间截断且不与按钮重叠。

点击按钮后显示现有删除确认框，确认框包含分区名称，并说明文件会移回桌面。用户取消时不修改文件、目录或配置。确认后执行现有安全删除流程。

按钮点击事件由控件直接处理，不触发分区移动或调整尺寸。按钮所属分区无需先成为当前选中分区，避免删除目标依赖菜单选择状态。

## 组件与事件流

1. `ZoneView` 创建和布局删除按钮，根据 `isEditing` 更新显隐状态，并上报自身分区 ID。
2. `ZoneWindow` 转发删除请求。
3. `WindowManager` 将请求转发给应用层。
4. `AppDelegate` 根据 ID 查找 `ZoneModel`，显示确认框并执行现有删除流程。
5. 删除成功后刷新分区窗口、文件缓存和状态栏菜单；失败时保留分区并显示现有错误提示。

应用层删除入口从依赖当前选择的菜单选择器调整为接收明确分区 ID 的内部方法。菜单重建逻辑保留新增和重命名操作，但不再创建删除菜单项。`ZoneEditMenuState` 不再负责删除菜单的启用状态。

## 错误处理与数据安全

- 找不到按钮上报的分区 ID 时直接返回，不修改状态。
- 文件恢复、目录删除或配置保存失败时沿用现有行为，不覆盖或静默删除用户文件。
- 日志继续包含分区名称和必要路径，不记录凭据。
- 删除按钮使用弱引用回调，避免视图、窗口和管理器之间形成引用环。

## 测试

应用层测试覆盖：

- 新建 `ZoneView` 后存在删除按钮。
- 普通模式下删除按钮隐藏，编辑模式下显示。
- 触发按钮会回调该视图对应的分区 ID。
- 事件经过 `ZoneWindow` 和 `WindowManager` 后仍保持正确分区 ID。
- 编辑菜单仍显示新增和重命名操作，但不再包含“删除当前分区…”。
- 标题绘制区域为删除按钮预留空间。

完成后运行：

```bash
swift test
swift build --product zonedesk-app
```

## 兼容性与风险

- 项目最低支持 macOS 12，系统图标选择该版本可用的 SF Symbol，并提供文本回退。
- 标题栏高度仅为 26 点，按钮布局必须保持可点击，同时不能遮挡圆角边框。
- AppKit 按钮会参与命中测试，需要验证点击不会落入 `ZoneView.mouseDown` 的移动逻辑。
