# Finder 风格分区文件选择与目录定位设计

## 目标

让分区中的文件具备接近 macOS Finder 的真实选择交互，并在分区标题栏右侧提供目录定位按钮。

## 范围

- 删除当前仅由鼠标悬停触发的整格灰色高亮。
- 单击文件时仅选中该文件。
- `Command + 单击` 切换单个文件的选中状态。
- `Shift + 单击` 从选择锚点到目标文件连续选择。
- 在空白区域拖动时显示选择框，并选中与选择框相交的文件。
- `Command + 拖动` 在原选择基础上追加框选结果。
- 单击空白区域清空选择。
- 双击文件继续调用现有打开文件行为。
- 选中文件时绘制 Finder 风格的图标选中框，以及使用系统强调色的文件名背景。
- 在分区标题栏右侧增加查找定位按钮；点击后在 Finder 中打开当前分区目录。

## 不在范围内

- 键盘方向键导航。
- 文件拖放、重命名或右键菜单。
- 跨分区共享选择状态。
- 像素级复制 Finder 的私有绘制实现。

## 设计

### 文件选择模型

选择状态由各自的 `ZoneFilesView` 管理，使用文件 URL 标识选中项，并记录最近一次普通选择的锚点索引。

- 普通单击替换当前选择，并更新锚点。
- `Command + 单击` 切换目标项，同时更新锚点。
- `Shift + 单击` 选择锚点和目标项之间的连续范围；没有锚点时退化为普通单击。
- 普通框选以拖动开始时的选择为空集；`Command` 框选以拖动开始时的选择为基础。
- 框选过程中，当前结果等于基础选择与所有和框选矩形相交的文件单元格的并集。
- 文件列表刷新后，删除已经不存在的选中 URL，并在锚点失效时清除锚点。

选择计算提取为不依赖事件对象的内部方法，使点击修饰键和矩形相交行为可以直接测试。

### 鼠标事件

鼠标按下时先判断是否命中文件单元格：

- 命中文件时按修饰键更新选择；双击仍打开该文件。
- 命中空白区域时进入潜在框选状态，并记录起点和拖动开始时的基础选择。
- 鼠标拖动超过系统拖动阈值后显示选择矩形并持续更新选中项。
- 鼠标抬起结束框选并保留最终选择。

取消原有 `mouseMoved` 驱动的文件 hover 高亮和手形光标，避免未选中项看起来像已选中。

### 选中外观

选中项分别绘制两个区域：

- 图标区域使用浅色半透明圆角背景和细边框。
- 文件名区域按实际标题区域绘制圆角系统强调色背景，文字保持白色。

颜色使用 AppKit 动态系统颜色，适应系统强调色及深浅模式。未选中项保持当前图标和白色标题显示。

### 分区目录定位按钮

`ZoneView` 在标题栏右侧放置一个无边框 AppKit 按钮，使用系统查找定位图标并提供中文工具提示。标题文本为按钮预留空间，避免重叠。

事件沿现有层级传递：

1. `ZoneView` 报告当前分区 ID。
2. `ZoneWindow` 转发分区 ID。
3. `WindowManager` 转发到应用层。
4. `AppDelegate` 根据当前配置取得 `ZoneModel`，调用 `ZoneLibrary.ensureDirectory(for:)`。
5. 使用 `NSWorkspace.shared.open(directoryURL)` 在 Finder 中直接打开该目录。

按钮在普通模式和编辑模式都显示。目录创建或打开失败时记录包含分区名称和路径的错误日志，不改变文件选择状态。

## 测试

应用层测试覆盖：

- 普通单击替换选择。
- `Command` 单击切换选择。
- `Shift` 单击连续选择。
- 空白点击清空选择。
- 框选选择相交单元格。
- `Command` 框选保留原选择。
- 文件列表刷新时移除失效选择。
- 双击仍回调正确文件 URL。
- 分区视图存在定位按钮，标题与按钮布局不重叠。
- 点击定位按钮沿窗口层级传出正确分区 ID。

核心层现有 `ZoneLibrary` 路径和目录创建测试继续作为目录 URL 的验证。最终运行完整 `swift test`。

## 兼容性与风险

- 项目仅声明 macOS 12，因此系统图标必须选择 macOS 12 可用的 SF Symbol，并提供文本回退。
- Finder 的私有选中绘制可能随系统版本变化，本实现只使用公开动态系统颜色模拟。
- 自绘视图需要确保框选与现有滚动事件不冲突；框选仅从文件视图空白区域开始。
